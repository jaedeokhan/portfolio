역할 : DB설계, UI설계, 로그인, 회원가입, 얼굴 데이터 수집, 얼굴 검출, 출석명단확인

MAPR기반 스파크 최적 성능 분석
개발기간 : 2019년 10월 01일 ~ 2019년 12월 20일
팀원 : 한재덕, 박현국, 김현
역할 : AWS EC2 환경구성, Ubuntu CLI를 통해서 Hadoop 설치, Spark 설치, Docker 설치, 성능 비교 및 분석, Python 시각화

특정 데이터 처리를 MAPR 기반 스파크(Spark)를 Host OS에서 적용한 환경과 MAPR기반 스파크를 도커라이징(Dockerizing)으로 구성한 환경을 비교해서 성능을 측정하고 성능은 벤치마킹 프로그램인 Intel사의 하이벤치(HiBench) 프로그램을 사용해서 측정했습니다.

주요 이슈로는 윈도우의 운영체제가 아닌 리눅스의 우분투를 사용하는 것이 익숙하지 않아서 명령어들을 사용해보고 손에 익히는데 시간이 꽤 걸렸습니다.
이때 사용한 명령어들을 빠르게 상기할 수 있게 구글클라우드에 핵심 내용을 정리해서 기록을 했습니다.
